(async) callBackendAPI(client, endpoint, data, method) → {Promise.<Object>}

🔗 백엔드 API 호출 핵심 함수

Freshdesk Custom App과 Python FastAPI 백엔드 간의 모든 통신을 처리하는 핵심 함수입니다. 멀티테넌트 환경에서 각 고객사의 Freshdesk 설정을 동적으로 적용하여 안전한 데이터 격리를 보장합니다.

주요 처리 과정:

  1. iparams에서 고객사별 Freshdesk 설정 로드
  2. 동적 헤더 구성 (X-Freshdesk-Domain, X-Freshdesk-API-Key)
  3. 백엔드 API 호출 및 응답 처리
  4. 에러 상황 시 적절한 폴백 처리
  5. 응답 데이터 검증 및 캐싱
Parameters:
Name Type Default Description
client Object

endpoint string

API 엔드포인트 (예: "init/123", "query")
data Object | null null

요청 데이터 (POST 요청의 경우)
method string GET

HTTP 메서드 ("GET" 또는 "POST")

// 자연어 쿼리 실행
const queryData = { query: "유사한 문제를 찾아줘", type: ["tickets", "solutions"] };
const result = await callBackendAPI(client, "query", queryData, "POST");

(async) validateAndInitializeApp() → {Promise.<boolean>}

🔍 앱 검증 및 초기화 메인 함수

앱 시작 시 모든 모듈의 의존성을 검증하고 초기화를 수행하는 핵심 함수입니다. 모듈 간 의존성 문제나 초기화 오류를 사전에 감지하여 안정적인 앱 실행을 보장합니다.

  1. 모든 필수 모듈(GlobalState, Utils, API, Data, UI, Events)의 로딩 상태 확인
  2. 각 모듈의 의존성 관계 검증 (예: UI는 GlobalState와 Utils에 의존)
  3. 검증 통과 시 메인 앱 초기화 진행
  4. 실패 시 사용자에게 친화적인 에러 메시지 표시
