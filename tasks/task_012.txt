# Task ID: 12
# Title: 프론트엔드 상태 관리 구현
# Status: pending
# Dependencies: 11
# Priority: high
# Description: BlockNote 에디터를 사용하여 LLM의 응답을 리치 텍스트 형식으로 표시하고, 필요한 경우 사용자의 입력을 받는 기능을 구현합니다.
# Details:
- `@blocknote/core` 및 `@blocknote/react` 라이브러리를 설치하고 프로젝트에 통합합니다.
- 'OO와 대화하기' 탭의 대화 내용 표시 영역에 BlockNote 에디터 컴포넌트를 렌더링합니다.
- 백엔드 `/query` API로부터 받은 LLM 응답(BlockNote JSON 형식 또는 마크다운)을 BlockNote 에디터가 이해할 수 있는 형식으로 변환하여 표시합니다.
  - 백엔드 응답이 마크다운인 경우, 이를 BlockNote 스키마로 변환하는 로직이 필요합니다. (예: `BlockNoteEditor.tryParseMarkdownToBlocks` 또는 유사 기능 활용)
  - 백엔드 응답이 이미 BlockNote JSON 형식이라면, 이를 직접 로드합니다.
- LLM 응답에 포함된 코드 블록, 목록, 인용구, 테이블 등의 리치 텍스트 요소가 올바르게 렌더링되도록 합니다.
- 사용자가 BlockNote 에디터에 직접 내용을 입력하는 기능은 초기에는 최소화하거나 읽기 전용으로 설정할 수 있으며, 추후 확장성을 고려합니다.
- 에디터의 스타일링을 Freshdesk UI와 일관되게 조정합니다.
- 대화가 길어질 경우 스크롤 처리 및 성능 최적화를 고려합니다.
- 모든 코드에는 상세한 한글 주석을 필수로 작성합니다.

# Test Strategy:
- BlockNote 에디터가 정상적으로 렌더링되는지 확인.
- 백엔드로부터 받은 다양한 형식의 LLM 응답(텍스트, 코드 블록, 목록 등)이 BlockNote 에디터에 정확히 표시되는지 검증.
- 마크다운-BlockNote 변환 로직(구현된 경우)의 정확성 테스트.
- 리치 텍스트 요소(코드 하이라이팅, 목록 들여쓰기 등)의 올바른 렌더링 확인.
- 에디터 스타일링 및 UI 일관성 검증.
- 긴 대화 내용에 대한 스크롤 및 성능 테스트.
